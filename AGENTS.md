# AGENTS.md — Hermes 공통 작업 규칙

이 문서는 AI(Hermes)가 이 프로젝트에서 작업할 때 반드시 지켜야 할 규칙입니다.

## 오늘 해도 되는 작업
- 프로젝트 파일 구조 읽기
- 사용 중인 프레임워크 확인
- 실행 명령 확인
- 문서(PROJECT_BRIEF.md, UI_REFERENCE.md, API_CONTRACT.md)와 이미지 분석
- 구현 계획 작성 (코드 수정 없이 계획만)
- 불명확한 내용 질문
- 기존 프로젝트 실행 점검

## 오늘 하면 안 되는 작업
- 앱 전체 자동 생성
- 로그인·DB·관리자 기능 전체를 한 번에 구현
- API를 추측해서 연결 (API_CONTRACT.md에 없는 API 임의 호출 금지)
- 프로젝트 전체 디자인 임의 변경
- 새 프레임워크 임의 설치
- 배포(Vercel 등) 임의 실행

## 진행 원칙
1. 새 기능은 반드시 "분석 → 질문 → 계획 → 사람 검토 → 구현" 순서로 진행한다.
2. 한 번에 하나의 작업 단위만 구현한다 (전체 자동 구현 금지).
3. UI_REFERENCE.md의 참고 이미지는 디자인 방향 참고용이며, 로고·브랜드명·가짜 데이터를 그대로 복사하지 않는다.
4. API_CONTRACT.md에 명시되지 않은 API 스펙은 추측하지 않고 반드시 확인을 요청한다.
5. API 키 등 민감 정보는 환경변수로만 관리하고 코드에 하드코딩하지 않는다.
6. 작업 완료 후에는 무엇을 수정했는지, 왜 수정했는지 간단히 보고한다.
